Rust · async · zero-copy

kova

Triangular arbitrage engine for cryptocurrency markets. Detect, evaluate, and execute — with built-in risk management.

Source Get Started
How it works

Three-leg cycle

Start with USDT, traverse three markets, return with more than you started.

1

Discover

Scan all exchange symbols. Build a graph and find every valid three-hop cycle back to your start asset.

2

Evaluate

Stream live order books via WebSocket. Walk bids and asks to compute effective prices, fees, and slippage.

3

Execute

Place three sequential market orders. Risk manager validates each leg. Automatic unwind on failure.

Features

Built for speed and safety

Async Rust, lock-free data structures, and multiple layers of risk protection.

Async Engine

Tokio-powered concurrent pipeline. WebSocket, detector, executor, and stats run as independent tasks.

Risk Management

Kill switch on consecutive losses or session loss limits. Min profit threshold, max trade size, and slippage buffer.

Paper Trading

Dry-run mode with live market data. Every opportunity logged to a JSONL journal queryable with DuckDB.

Multi-Exchange

Generic Exchange trait with static dispatch. Currently supports Binance and Paribu. Add new exchanges in a module.

Auto Discovery

Fetches all exchange symbols, builds an adjacency graph, and finds every triangle path. No manual configuration.

Live Analysis

Periodic stats (win rate, PnL, evaluations). Post-session analysis via kova analyze or DuckDB.

Usage

CLI commands

Six subcommands. From discovery to post-trade analysis.

discover

Find Triangles

Scan all exchange symbols and list valid triangle paths for a start asset.

kova discover --start-asset USDT
dry-run

Paper Trade

Stream live data, detect opportunities, log theoretical trades — no real orders.

kova dry-run --config kova.toml
run

Live Trading

Execute real trades through the full pipeline with risk management enabled.

kova run --config kova.toml
analyze

Review Results

Parse the JSONL trade journal and compute win rate, PnL, and top paths.

kova analyze --journal kova-journal.jsonl
Install

Build from source

Requires Rust 1.75+. Clone, build, configure, run.

# Clone and build
$ git clone https://github.com/selimozten/kova.git
$ cd kova
$ cargo build --release

# Configure
$ cp kova.toml.example kova.toml
$ $EDITOR kova.toml  # set api_key and api_secret

# Paper trade
$ ./target/release/kova dry-run
Configuration

kova.toml

TOML config with sensible defaults. Only API keys are required.

# kova.toml
[general]
start_asset = "USDT"
trade_amount = 100.0
dry_run = true

[exchange]
name = "binance"
api_key = "YOUR_KEY"
api_secret = "YOUR_SECRET"

[risk]
min_profit_pct = 0.15      # minimum 0.15% profit to trigger
max_trade_size = 500.0     # cap per trade
max_session_loss = 50.0    # kill switch threshold
slippage_buffer_bps = 2.0 # basis points buffer
cooldown_ms = 500          # ms between trades
